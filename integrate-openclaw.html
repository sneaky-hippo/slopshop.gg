Integrate Slopshop with OpenClaw

OpenClaw is the leading open-source agent framework. Add Slopshop as the brain layer via MCP for verifiable memory, 1,303 tools, and secretless credentials — no infrastructure changes required.

Prerequisites
Node.js 18+, an OpenClaw project, and a free Slopshop account. The MCP integration works with OpenClaw v2.x and later.
1Install the Slopshop CLI

Install the CLI globally and create your account. The CLI manages your API keys, vault secrets, and MCP server lifecycle.

npm install -g slopshop
slop signup

After signup, your API key is stored at ~/.slop/credentials. You can also retrieve it from the dashboard at slopshop.gg/dashboard.

Verify the installation:

slop --version
slop status
2Configure OpenClaw MCP

OpenClaw uses an MCP config file to discover and connect to tool servers. Add the Slopshop MCP server entry to your OpenClaw configuration. The config file is typically located at .openclaw/mcp.json in your project root, or globally at ~/.openclaw/mcp.json.

{
  "mcpServers": {
    "slopshop": {
      "command": "slop",
      "args": ["mcp", "serve"],
      "env": {
        "SLOP_API_KEY": "sk-slop-your-key-here"
      }
    }
  }
}
Replace sk-slop-your-key-here with your actual key from slop status or the dashboard. Never commit API keys to source control — use environment variables or the vault pattern in Step 4.

Restart OpenClaw after saving the config. You should see slopshop appear in the connected MCP servers list in the OpenClaw UI or CLI output.

3Verifiable Memory in OpenClaw Skills

Every memory write returns a proof_hash and merkle_root — cryptographic receipts you can store, share, or audit later. Use the memory-set and memory-get MCP tools inside any OpenClaw skill.

Writing memory with a proof

{
  "tool": "memory-set",
  "input": {
    "key": "research/findings/2024-Q4",
    "value": {
      "summary": "Market analysis complete",
      "sources": ["arxiv:2401.00001", "arxiv:2401.00042"],
      "confidence": 0.94
    },
    "namespace": "openclaw-agent"
  }
}

The response includes a verifiable proof you can log or pass downstream:

{
  "status": "stored",
  "key": "research/findings/2024-Q4",
  "proof_hash": "sha256:a3f9c2e1b8d47...",
  "merkle_root": "0x7a2b4f...",
  "timestamp": "2024-01-15T10:23:44Z"
}

Reading memory back

{
  "tool": "memory-get",
  "input": {
    "key": "research/findings/2024-Q4",
    "namespace": "openclaw-agent"
  }
}

Semantic search across memories

{
  "tool": "memory-search",
  "input": {
    "query": "market analysis confidence high",
    "namespace": "openclaw-agent",
    "limit": 5
  }
}
4Secretless Vault Pattern

Stop putting API keys in your skill code. Store them once in the Slopshop vault (AES-256-GCM encrypted), then reference the vault_id in your OpenClaw skills. The key is never returned in plaintext — only used server-side for proxied requests.

Store a secret via CLI

slop vault set GITHUB_TOKEN ghp_your_token_here
# output: vault_id = vlt_7f3a9c2e1b8d

Use the vault in a skill

{
  "tool": "vault-proxy",
  "input": {
    "vault_id": "vlt_7f3a9c2e1b8d",
    "url": "https://api.github.com/user/repos",
    "method": "GET"
  }
}

The Slopshop backend injects the credential, makes the request, and returns the result. Your skill code never touches the raw token.

Troubleshooting

MCP server not appearing in OpenClaw

Ensure the CLI is on your PATH: which slop should return a path. If it doesn't, check your npm global bin directory is included in $PATH. Restart OpenClaw completely after editing the MCP config.

Authentication errors (401)

Verify your key with slop status. Keys starting with sk-slop- are valid. If you regenerated your key in the dashboard, update the value in mcp.json and restart OpenClaw.

Memory writes succeed but reads return null

Check that the namespace parameter matches between your write and read calls. Namespaces are case-sensitive and default to "default" if omitted.

Slow tool response times

The MCP server starts a new process per session. For high-throughput scenarios, run slop mcp serve --persistent to keep the server alive between OpenClaw sessions and reduce cold-start latency.

Vault proxy returning 403

The vault_id must belong to the same account as your API key. List your vault entries with slop vault list to confirm the ID. Domain-restricted vault entries will only proxy to allowed hostnames — check restrictions with slop vault inspect <vault_id>.
