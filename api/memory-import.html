Import Memories from JSON API

Import a set of memories from a JSON object (as produced by memory-export). Supports merge mode (preserve existing keys) or overwrite mode. Returns count of imported, skipped, and overwritten entries. Available as a simple REST API — POST your input, get structured JSON back. 1 credit per call, no rate limiting, works in any language with HTTP support.

Try it live →

How it works

POST a JSON object with your input data to /v1/memory-import. The API processes your request server-side using real computation (not mocked data) and returns a structured JSON response with your results and metadata including credits used and processing time.

Use cases

  • Use Import Memories from JSON in automated pipelines without installing a library
  • Call Import Memories from JSON from AI agent workflows via MCP or REST
  • Integrate Import Memories from JSON into serverless functions with zero dependencies
  • Build Import Memories from JSON functionality into any language that supports HTTP

API Reference

POST https://slopshop.gg/v1/memory-import
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

Examples

Three real-world scenarios showing how developers use Import Memories from JSON in production.

Example 1
Store a simple value
Persist a string value for retrieval in a future session.
curl -X POST https://slopshop.gg/v1/memory-import \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"key": "my_key", "value": "hello world"}'
Example 2
Store structured data
Save a JSON object as agent state.
curl -X POST https://slopshop.gg/v1/memory-import \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"key": "user_preferences", "value": {"theme": "dark", "language": "en", "notifications": true}}'
Example 3
Store with namespace
Save a value under a specific namespace for isolation.
curl -X POST https://slopshop.gg/v1/memory-import \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"key": "session_data", "value": {"step": 2, "completed": true}, "namespace": "agent:session:xyz"}'

Related APIs in Memory

  • Store Memory API — Store a named memory with a string or JSON value, optional tags for organization, and an optional namespace for isolation. Persists across agent sessions. Returns confirmation with storage timestamp.
  • Retrieve Memory API — Retrieve a stored memory by its key within an optional namespace. Returns the value, creation timestamp, last-updated timestamp, and associated tags. Returns null if key does not exist.
  • Search Memories API — Search stored memories by tag match, key substring, or value substring within an optional namespace. Returns all matching key-value pairs with their metadata, sorted by last-updated date.
  • List All Memories API — List all stored memory keys in a namespace, with optional filtering by tag. Returns key names, creation dates, value sizes, and tags. Does not return values themselves (use memory-get for that).
  • Delete Memory API — Delete a stored memory by key within an optional namespace. Returns confirmation including the deleted key and a timestamp. Silently succeeds if key does not exist.
