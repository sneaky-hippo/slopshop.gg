Fetch and Parse Sitemap API

Fetch a domain's sitemap.xml (or sitemap index), parse it, and return all listed URLs with their last-modified dates and change frequencies. Available as a simple REST API — POST your input, get structured JSON back. 3 credits per call, no rate limiting, works in any language with HTTP support.

Try it live →

How it works

POST a JSON object with your input data to /v1/sense-url-sitemap. The API processes your request server-side using real computation (not mocked data) and returns a structured JSON response with your results and metadata including credits used and processing time.

Use cases

  • Use Fetch and Parse Sitemap in automated pipelines without installing a library
  • Call Fetch and Parse Sitemap from AI agent workflows via MCP or REST
  • Integrate Fetch and Parse Sitemap into serverless functions with zero dependencies
  • Build Fetch and Parse Sitemap functionality into any language that supports HTTP

API Reference

Examples

Three real-world scenarios showing how developers use Fetch and Parse Sitemap in production.

Example 1
Query a production domain
Look up fetch and parse sitemap data for a live production domain.
curl -X POST https://slopshop.gg/v1/sense-url-sitemap \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "example.com"}'
Example 2
Check an API endpoint domain
Run fetch and parse sitemap on an external API host.
curl -X POST https://slopshop.gg/v1/sense-url-sitemap \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "api.github.com"}'
Example 3
Monitor your own infrastructure
Check fetch and parse sitemap for your application domain.
curl -X POST https://slopshop.gg/v1/sense-url-sitemap \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "app.yourproduct.com"}'

Related APIs in Sense: Web

  • Fetch URL as Clean Text API — Fetch a URL and return clean readable text content, stripping all HTML tags, scripts, and styles. Returns the main textual content a human would read.
  • Extract Links from URL API — Fetch a URL and extract all hyperlinks (href attributes), returning absolute URLs with their anchor text and whether they are internal or external.
  • Get URL Metadata API — Fetch a URL and extract meta information: page title, meta description, Open Graph tags (og:title, og:image, og:description), Twitter card tags, and canonical URL.
  • Detect Website Tech Stack API — Fetch a URL and detect technologies in use: JavaScript frameworks (React, Vue, Angular), CMS generators (WordPress, Ghost), analytics tools, CDNs, and server software from headers and HTML patterns.
  • Measure URL Response Time API — Make multiple HTTP requests to a URL and return min, max, and average response times in milliseconds. Useful for detecting latency patterns and slowdowns.
