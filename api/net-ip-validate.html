IP Validate API

Validate IP address, detect version (v4/v6), check if private. Available as a simple REST API — POST your input, get structured JSON back. 1 credit per call, no rate limiting, works in any language with HTTP support.

Try it live →

How it works

POST a JSON object with your input data to /v1/net-ip-validate. The API processes your request server-side using real computation (not mocked data) and returns a structured JSON response with your results and metadata including credits used and processing time.

Use cases

  • Use IP Validate in automated pipelines without installing a library
  • Call IP Validate from AI agent workflows via MCP or REST
  • Integrate IP Validate into serverless functions with zero dependencies
  • Build IP Validate functionality into any language that supports HTTP

API Reference

POST https://slopshop.gg/v1/net-ip-validate
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

Examples

Three real-world scenarios showing how developers use IP Validate in production.

Example 1
Query a production domain
Look up ip validate data for a live production domain.
curl -X POST https://slopshop.gg/v1/net-ip-validate \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "example.com"}'
Example 2
Check an API endpoint domain
Run ip validate on an external API host.
curl -X POST https://slopshop.gg/v1/net-ip-validate \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "api.github.com"}'
Example 3
Monitor your own infrastructure
Check ip validate for your application domain.
curl -X POST https://slopshop.gg/v1/net-ip-validate \
  -H "Authorization: Bearer $SLOPSHOP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"domain": "app.yourproduct.com"}'
