Migrate from Mem0 to Slopshop

Mem0 locks your agent memory to AWS infrastructure with no verifiability and a limited free tier. Slopshop is self-hostable, gives every write a cryptographic proof, and keeps core memory always free.

Migration takes under 10 minutes. The script below exports all your Mem0 memories and imports them into Slopshop with verifiable proofs automatically.

API Mapping

Every Mem0 operation has a direct Slopshop equivalent. Swap the calls and you're done — no logic changes needed.

Mem0Slopshop
m.add(messages, user_id) POST /v1/memory-set {"key": ..., "value": ...}
m.search(query) POST /v1/memory-search {"query": ...}
m.get_all(user_id) POST /v1/memory-list {}
m.delete(memory_id) POST /v1/memory-delete {"key": ...}
m.update(memory_id, data) POST /v1/memory-set (same key overwrites)
m.get(memory_id) POST /v1/memory-get {"key": ...}

1Get Your Slopshop Key

Install the CLI and create your free account. Your key is automatically saved to ~/.slop/credentials.

npm install -g slopshop
slop signup
slop status  # prints your sk-slop-... key
2Run the Migration Script

This script exports all memories from your Mem0 account and imports them into Slopshop. Each imported memory gets a proof_hash for tamper-evidence. Save it as migrate_mem0.py and run it once.

migrate_mem0.py
#!/usr/bin/env python3
"""Migrate memories from Mem0 to Slopshop"""
import requests

def migrate(mem0_client, slop_key, namespace="migrated"):
    slop = "https://slopshop.gg"
    headers = {"Authorization": f"Bearer {slop_key}"}

    memories = mem0_client.get_all()
    migrated = 0

    for mem in memories:
        key = mem.get("id", f"mem_{migrated}")
        value = mem.get("memory", str(mem))

        r = requests.post(f"{slop}/v1/memory-set",
            headers=headers,
            json={"key": key, "value": value, "namespace": namespace})

        result = r.json()
        if result.get("status") == "stored":
            print(f"  Migrated: {key} → proof_hash: {result.get('proof_hash', '')[:16]}...")
            migrated += 1
        else:
            print(f"  FAILED:   {key} → {result.get('error', 'unknown error')}")

    print(f"\nMigration complete: {migrated}/{len(memories)} memories")
    print("Each memory now has a cryptographic proof. Run 'slop proof verify' to verify.")

if __name__ == "__main__":
    import sys
    slop_key = sys.argv[1] if len(sys.argv) > 1 else input("Slopshop API key: ")
    print("Connect your Mem0 client and call: migrate(mem0_client, slop_key)")

3Update Your Agent Code

Replace Mem0 imports with direct Slopshop calls using the API mapping table above. The namespace "migrated" used during import will contain all your memories — update your recall calls to use it, or migrate to a new namespace per agent.

import requests

SLOP_KEY = "sk-slop-your-key-here"
SLOP_BASE = "https://slopshop.gg"
HEADERS = {"Authorization": f"Bearer {SLOP_KEY}"}

def remember(key, value, namespace="default"):
    return requests.post(f"{SLOP_BASE}/v1/memory-set",
        headers=HEADERS,
        json={"key": key, "value": value, "namespace": namespace}).json()

def recall(key, namespace="default"):
    return requests.post(f"{SLOP_BASE}/v1/memory-get",
        headers=HEADERS,
        json={"key": key, "namespace": namespace}).json().get("value")

def search(query, namespace="default", limit=10):
    return requests.post(f"{SLOP_BASE}/v1/memory-search",
        headers=HEADERS,
        json={"query": query, "namespace": namespace, "limit": limit}).json().get("results", [])
4Verify Your Proofs (Optional)

Every imported memory has a proof_hash. You can verify any of them at any time to confirm the data hasn't changed since import:

slop proof verify --hash sha256:a3f9c2e1b8d47...
# verified: key=usr_abc123, namespace=migrated
# stored: 2024-01-15T10:23:44Z
# status: untampered

Or verify the full namespace batch:

slop proof verify-namespace --namespace migrated
# 847/847 memories verified, 0 tampered

Self-Hosting Slopshop

Unlike Mem0, you can run the entire Slopshop backend on your own infrastructure. Everything your agents store stays on your servers.

git clone https://github.com/slopshop/slopshop
cd slopshop
npm install
cp .env.example .env  # add your OPENAI_API_KEY or Ollama URL
node server-v2.js

Your local server accepts the same REST API as the cloud version. Point SLOP_BASE to http://localhost:3000 in your agent code and nothing else changes.

Ollama support
Slopshop supports local Ollama models for all memory operations — no OpenAI API key required for self-hosted setups. Set OLLAMA_BASE_URL=http://localhost:11434 in your .env file.
