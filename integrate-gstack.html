Integrate Slopshop with GStack

GStack specializes in multi-role agent workflows. Slopshop adds persistent verifiable memory and tool access to any GStack agent via a simple REST API — no SDK required, just requests.

Prerequisites
Python 3.8+, a GStack project with at least one agent role, and a free Slopshop account. Install dependencies: pip install requests

Quick Setup

Drop this helper module into your GStack project. All three primitives you need — remember(), recall(), and vault_proxy() — are covered in under 30 lines of Python.

import requests

SLOP_KEY = "sk-slop-your-key-here"
SLOP_BASE = "https://slopshop.gg"

def remember(key, value, namespace="gstack"):
    r = requests.post(f"{SLOP_BASE}/v1/memory-set",
        headers={"Authorization": f"Bearer {SLOP_KEY}"},
        json={"key": key, "value": value, "namespace": namespace})
    d = r.json()
    return d  # includes proof_hash and merkle_root

def recall(key, namespace="gstack"):
    r = requests.post(f"{SLOP_BASE}/v1/memory-get",
        headers={"Authorization": f"Bearer {SLOP_KEY}"},
        json={"key": key, "namespace": namespace})
    return r.json().get("value")

def vault_proxy(vault_id, url, method="GET", body=None):
    r = requests.post(f"{SLOP_BASE}/v1/vault/proxy",
        headers={"Authorization": f"Bearer {SLOP_KEY}"},
        json={"vault_id": vault_id, "url": url, "method": method, "body": body})
    return r.json()

Swap SLOP_KEY for an environment variable in production: os.environ["SLOP_API_KEY"]. Get your key from slop status or the dashboard.

Agent Integration Pattern

The canonical pattern for GStack multi-role agents: each agent recalls shared context before starting, and remembers its findings after completing its step. This gives every subsequent agent in the workflow full context without passing state through orchestrator messages.

import gstack
from slop_helpers import remember, recall  # the module above

class ResearchAgent(gstack.Agent):
    role = "researcher"

    def run(self, task):
        # Recall what previous agents have already found
        prior_context = recall("workflow/shared-context", namespace="gstack")
        if prior_context:
            task = f"{task}\n\nPrior context: {prior_context}"

        # Do the actual research work
        result = self.llm.complete(task)

        # Persist findings for downstream agents
        proof = remember("research/latest-findings", {
            "summary": result.text,
            "agent": self.role,
            "timestamp": result.created_at
        }, namespace="gstack")

        # Log proof for audit trail
        print(f"[audit] proof_hash={proof['proof_hash'][:24]}...")
        return result


class WriterAgent(gstack.Agent):
    role = "writer"

    def run(self, task):
        # Pick up exactly what researcher stored
        findings = recall("research/latest-findings", namespace="gstack")
        draft = self.llm.complete(f"Write based on: {findings}")

        remember("writer/draft", draft.text, namespace="gstack")
        return draft

Verifiable Memory Pattern

Every memory-set response includes a proof_hash (SHA-256 of the stored value) and a merkle_root anchored to the Slopshop audit chain. Store these in your workflow logs to create a tamper-evident record of what each agent knew and when.

def remember_with_audit(key, value, namespace="gstack"):
    """Store memory and write proof to local audit log."""
    result = remember(key, value, namespace)

    if result.get("status") == "stored":
        with open("agent_audit.log", "a") as f:
            f.write(
                f"{result['timestamp']} | key={key} | "
                f"proof={result['proof_hash']} | "
                f"root={result['merkle_root']}\n"
            )

    return result


# Later, verify any stored value hasn't changed:
# slop proof verify --hash sha256:a3f9c2e1b8d47...

To verify proofs from the CLI at any time:

slop proof verify --hash sha256:a3f9c2e1b8d47...
# verified: key=research/latest-findings, stored 2024-01-15T10:23:44Z, untampered

Vault Pattern for External APIs

GStack agents often need to call external APIs (GitHub, Notion, Slack, etc.). Instead of embedding API keys in your agent code or GStack environment config, store them in the Slopshop vault once and reference by vault_id. The raw credential never leaves Slopshop servers.

Store credentials via CLI

slop vault set GITHUB_TOKEN ghp_your_token_here
# vault_id: vlt_7f3a9c2e1b8d

slop vault set NOTION_KEY secret_your_notion_key
# vault_id: vlt_2b9e4f1a3c7d

Use in a GStack agent

class GithubAgent(gstack.Agent):
    role = "github-integrator"

    VAULT_GITHUB = "vlt_7f3a9c2e1b8d"

    def list_repos(self, org):
        # Credential injected server-side — never in this code
        response = vault_proxy(
            vault_id=self.VAULT_GITHUB,
            url=f"https://api.github.com/orgs/{org}/repos",
            method="GET"
        )
        repos = response.get("body", [])
        remember(f"github/{org}/repos", repos, namespace="gstack")
        return repos

    def create_issue(self, repo, title, body):
        return vault_proxy(
            vault_id=self.VAULT_GITHUB,
            url=f"https://api.github.com/repos/{repo}/issues",
            method="POST",
            body={"title": title, "body": body}
        )

REST API Reference

All endpoints are POST to https://slopshop.gg with Authorization: Bearer <key> and Content-Type: application/json.

ENDPOINTS
POST /v1/memory-set Store a value with verifiable proof
POST /v1/memory-get Retrieve a stored value by key
POST /v1/memory-search Semantic search across a namespace
POST /v1/memory-list List all keys in a namespace
POST /v1/memory-delete Delete a memory by key
POST /v1/vault/proxy Proxy a request using a vault credential
ParameterTypeRequiredDescription
keystringYes (set/get/delete)Unique key within the namespace
valueanyYes (set)Value to store — string, object, or array
namespacestringNoLogical partition, defaults to "default"
querystringYes (search)Natural language search query
limitintegerNoMax results for search, defaults to 10
vault_idstringYes (proxy)ID from slop vault set
