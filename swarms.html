How it works

Three steps to a live swarm

Define your agents, fire the orchestrator, let Dream consolidate the output overnight. Every run produces a scored result set you can query, replay, or merge.

01

Define agents

Create a swarm config with POST /v1/swarm/create. Declare each agent's role, model, memory namespace, and tool permissions. Agents are typed: researcher, analyst, synthesizer, or custom.

02

Orchestrate

Call POST /v1/swarm/orchestrate with your goal. The engine fans out tasks to each agent in parallel or in dependency order, applies your merge policy, and returns a 6-axis score on the consolidated result.

03

Dream consolidate

Pass dream_after: true to queue a Dream Engine session after orchestration completes. Swarm outputs flow into REM-cycle synthesis — patterns, insights, and compressed memory written back to each agent's namespace.

Scoring

6-axis result scoring

Every orchestration run returns a score object. Six axes, each 0.0–1.0. Thresholds are configurable — fail fast, retry, or escalate to human-in-loop based on any combination.

SuccessProb
success_prob

Estimated probability the swarm's output correctly satisfies the stated goal. Computed from task completion signals, agent confidence scores, and cross-agent consistency checks.

range: 0.0 – 1.0  ·  threshold key: min_success_prob
Robustness
robustness

Measures consistency of outputs across agents. High robustness means multiple independent agents reached similar conclusions. Low robustness surfaces conflicts that may require human review.

range: 0.0 – 1.0  ·  threshold key: min_robustness
Foresight
foresight

Captures how well the swarm anticipated downstream consequences and edge cases in its reasoning. Derived from chain-of-thought depth and counter-hypothesis coverage.

range: 0.0 – 1.0  ·  threshold key: min_foresight
GoalAlignment
goal_alignment

Semantic similarity between the original goal and each agent's output, averaged and weighted by agent role. Catches goal drift in long chains. Low values trigger automatic re-routing.

range: 0.0 – 1.0  ·  threshold key: min_goal_alignment
Efficiency
efficiency

Ratio of useful output tokens to total tokens consumed across the swarm. High efficiency means agents stayed on-task. Low efficiency flags redundant reasoning or off-topic detours.

range: 0.0 – 1.0  ·  threshold key: min_efficiency
Cost
cost

Normalized credit spend relative to a budget you set in the swarm config. Score of 1.0 means on-budget. Below 0.5 means you're over 2x budget. Set max_credits to hard-cap the run.

range: 0.0 – 1.0  ·  config key: max_credits
API Reference

Endpoints

All endpoints accept and return JSON. Authentication via X-API-Key header or Authorization: Bearer <key>.

Method Path Description
POST
/v1/swarm/create
Define agents, roles, model assignments, memory namespaces, merge policy, and scoring thresholds. Returns a swarm_id.
 Create swarm config
POST
/v1/swarm/orchestrate
Run orchestration against a goal. Pass swarm_id, goal, optional context, dream_after, and score_thresholds. Blocks or streams depending on async flag.
 Run orchestration + 6-axis scoring
GET
/v1/swarm/:id/status
Poll run status for async orchestrations. Returns phase (pending, running, dreaming, done, failed), partial scores, and per-agent progress.

GET
/v1/swarms
List all swarm configs and recent run summaries for your API key. Supports ?status=, ?limit=, and ?cursor= for pagination.
 List all swarms

Merge Policies

How outputs get combined

Set merge_policy in your swarm config to control how agent outputs are reconciled into a final result.

Auto
auto

The engine picks the best merge strategy based on agent count, output types, and role topology. Good default for most workflows. Uses structural diffing for structured outputs and LLM merge for prose.

LLM Smart
llm-smart

A dedicated merge agent reads all agent outputs and produces a single coherent result. Best for prose, research briefs, and multi-perspective analysis. Adds one extra LLM call and ~20 credits.

Human in Loop
human-in-loop

Orchestration pauses and surfaces a diff of conflicting outputs to a webhook or UI callback. The human approves, rejects, or edits before the run continues. Combine with min_robustness thresholds for surgical review.

Agent Debate
agent-debate

Agents critique each other's outputs in a structured debate round. Each agent sees the others' results and can revise its own. Converges to a consensus after N rounds (configurable via debate_rounds). Highest quality, highest cost.

Dream Integration

Consolidate results overnight

Swarms generate a lot of intermediate reasoning. Dream Engine turns that raw output into durable, compressed, searchable memory — automatically.

☾
dream_after: true

Set dream_after: true in your orchestration request and Slopshop queues a Dream Engine job the moment the swarm completes. The job runs all five REM strategies — synthesize, pattern_extract, insight_generate, compress, and associate — over each agent's memory namespace.

When your agents wake up for the next run, they have a consolidated memory of everything the swarm produced. No manual summarization. No prompt stuffing. The knowledge is already there, structured and queryable via POST /v1/memory/search.

Track the dream job with the dream_job_id returned in every orchestration response. Poll it via GET /v1/memory/dream/status/:id.

→  Tip: schedule overnight dreams with POST /v1/schedule using a cron expression. Your swarm can run at 2am and wake up with fresh memory every morning.
