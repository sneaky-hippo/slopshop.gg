What Slopshop does

Everything your agent needs after the prompt

Most agents can generate text. Few can use tools, remember context, schedule work, or run multi-step workflows without glue code.

Tools

1,303 handlers across 82 categories. Crypto, text, math, validation, finance, DevOps, AI utilities, network, code execution. Every handler benchmarked, tested, p95 <1ms.

slop call crypto-hash-sha256 --data hello curl -X POST slopshop.gg/v1/crypto-hash-sha256 \ -H "Authorization: Bearer $KEY" -d '{"data":"hello"}'
Persistent Memory

SQLite-backed. Key-value, queues, counters, namespaces, semantic search. Cross-LLM, cross-session. Core memory APIs are always free — 0 credits.

POST /v1/memory-set {"namespace":"project","key":"findings","value":"React + Next.js"} → {"status":"stored","credits_used":0}
Execution

Sandboxed code execution, batch processing, workflow chaining, parallel fan-out. Run agent-generated code safely, chain tool calls, process results at scale.

POST /v1/batch {"calls":[ {"slug":"crypto-hash-sha256","body":{"data":"a"}}, {"slug":"crypto-hash-sha256","body":{"data":"b"}} ]}
Scheduling

Cron-style recurring execution with webhook callbacks. Prompt queues for deferred workloads. Intervals from 1 minute to 7 days. Your agent runs on schedule, not just on demand.

POST /v1/schedules {"slug":"sense-url-tech-stack","interval":"6h", "body":{"url":"https://competitor.com"}}

Frequently asked questions

What is Slopshop?

A backend platform for AI agents. It provides tools (1,303 handlers across 82 categories), persistent memory (free forever), scheduling (cron + webhooks), and execution (sandboxed code, batch processing, workflow chaining) in one place. Use it from MCP, API, or CLI.

Who is Slopshop for?

Developers building AI agents that need to do more than generate text. Internal tools teams replacing backend glue. Anyone who needs to give their agent persistent memory, real compute, or scheduled execution.

Is MCP required?

No. Slopshop works as an MCP server (for Claude Code, Cursor, Goose, etc.), a REST API (for any backend), or a standalone CLI (slop call <tool>). Use whichever interface fits your workflow.

Can I start small and expand later?

Yes. Start with one tool call from the CLI. Add memory when you need persistence. Add scheduling when you need automation. Add batch processing when you need throughput. Everything is incremental — no upfront commitment.

What is included in the free tier?

500 free credits on signup (no credit card). Full access to all 1,303 handlers. Core memory APIs (set, get, search, list, delete, stats) are always free — 0 credits forever.

How does Slopshop handle security?

CSP enabled, timing-safe auth, memory isolation per API key, HTTPS enforced, input validation on every endpoint, sandboxed code execution (vm.createContext), rate limiting with Retry-After headers. Self-hostable for full control. Full security details →
