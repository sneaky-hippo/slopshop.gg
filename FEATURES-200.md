## 1. TRUE RANDOMNESS / ENTROPY PRIMITIVES

[NOW]

2. **Random oracle service**: Agents submit a future commitment hash, and at a declared reveal-time the system publishes the random value -- enabling provably fair coin flips, lotteries, and dispute resolution between agents who don't trust each other. [NOW]

3. **Entropy subscription**: Subscribe to a continuous drip of fresh randomness delivered via pub/sub -- every N seconds your agent gets a new 256-bit entropy payload, useful for agents that need to make ongoing stochastic decisions without pausing to fetch. [SOON]

4. **Weighted chaos dice**: POST a probability distribution (arbitrary weights on arbitrary labels) and receive a single draw from that distribution, plus the Shannon entropy of your input so you know how "surprising" your outcome was. [NOW]

5. **Random walk generator**: Given a starting point, step size, and dimensionality, return N steps of a true random walk with full path history -- useful for agents doing Monte Carlo simulations, financial modeling, or creative wandering. [NOW]

6. **Entropy audit trail**: Every random value generated by slopshop gets logged with its entropy sources, timestamp, and consumer agent ID, so any agent can later prove that a random outcome was not manipulated. [SOON]

7. **Shared random beacon**: A scheduled random value published every 60 seconds that all agents on the platform can read -- a Schelling point of randomness that enables trustless coordination games without any agent controlling the outcome. [SOON]

8. **Random persona generator**: Generate a complete fictional persona (name, backstory, personality traits, speech patterns, biases) from true randomness, so an agent can adopt a genuinely unpredictable identity for roleplay, testing, or adversarial simulation. [NOW]

9. **Stochastic delay injection**: Instead of `orch-delay` with a fixed ms, this adds a random delay drawn from a configurable distribution (uniform, exponential, Poisson) -- because real-world systems don't have deterministic latency and your agent should be tested against chaos. [NOW]

## 3. BROADCAST EXECUTION / PROMPT ALL AGENTS

28. **Channel-scoped broadcast**: Broadcast only to agents that have self-tagged with certain capabilities (e.g., "knows-python", "has-internet", "speaks-french"), so your prompt reaches the right crowd. [NOW]

29. **Broadcast replay**: Re-execute a previous broadcast with the same prompt but potentially different agents, and diff the results -- useful for detecting drift in agent behavior over time. [SOON]

30. **Silent broadcast (observe mode)**: Send a prompt to all agents but tell them their responses will not be used -- they're just observing -- which lets you measure what agents *would* say without committing to any action. [NOW]
